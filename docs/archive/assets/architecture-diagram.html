2. Datenfluss Frontend zu Backend
sequenceDiagram
    autonumber
    actor User as Benutzer
    participant FE as React Frontend
    participant RQ as React Query
    participant AW as Appwrite SDK
    participant AuthS as Appwrite Auth
    participant DB as Appwrite Database
    participant Fn as Functions
    participant Job as scriptony-jobs
    participant AI as scriptony-ai
    participant Ext as Externer AI

    User->>FE: Login
    FE->>AW: createEmailPasswordSession()
    AW->>AuthS: Auth Request
    AuthS-->>AW: Session + JWT
    AW-->>FE: Session
    FE->>RQ: invalidateQueries()

    User->>FE: Projekt erstellen
    FE->>RQ: mutation
    RQ->>AW: createDocument()
    AW->>DB: Write
    DB-->>AW: Document
    AW-->>RQ: Response
    RQ-->>FE: UI Update

    User->>FE: Beat generieren
    FE->>Fn: POST scriptony-ai
    Fn->>Job: createJob
    Job->>DB: Job Document
    Job-->>Fn: Job ID
    Fn-->>FE: Job ID

    loop Polling
        FE->>RQ: getJobStatus
        RQ->>AW: getDocument
        AW->>DB: Read
        DB-->>AW: Status
        AW-->>RQ: Status
    end

    Job->>AI: processJob
    AI->>Ext: API Call
    Ext-->>AI: Response
    AI->>DB: Update Job
    AI-->>Job: Done
    RQ-->>FE: Completed
    FE->>User: Ergebnis

    User->>FE: Audio upload
    FE->>AW: createFile
    AW->>DB: Bucket Write
    DB-->>AW: File ID
    AW-->>FE: Response
    FE->>RQ: Refetch
